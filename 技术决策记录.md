# 技术决策记录 (ADR)

## ADR-001: Session 超时时间调整

### 状态
已接受

### 上下文
项目原有的 Session 超时时间设定为 60 分钟（3600 秒）。为了提升用户体验，特别是应对长时间在线刷题的需求，用户要求将超时时间延长至 100 分钟（6000 秒）。

### 决策
我们将通过以下方式实现 Session 超时时间的调整：
1. 修改 `inc/functions.inc.php` 中的核心 Session 启动函数（`startStudentSession` 和 `startAdminSession`），显式调用 `ini_set` 设置 `session.gc_maxlifetime` 和 `session.cookie_lifetime`。
2. 同步更新项目根目录下的 `.user.ini` 配置文件，作为 PHP 引擎层面的全局项目配置。

### 方案对比

| 维度 | 方案 1: PHP 全局配置 | 方案 2: .user.ini 局部配置 | 方案 3: 代码动态设置 (ini_set) |
| :--- | :--- | :--- | :--- |
| **性能** | 最高 | 中 | 中 |
| **可维护性** | 低（系统级耦合） | 中（环境相关） | 高（代码控制） |
| **扩展性** | 差 | 一般 | 强（可区分角色） |
| **结论** | 不采用 | **采用（作为保障）** | **主要采用** |

### 后果
- 用户在无操作的情况下，Session 将保持 100 分钟有效。
- 服务器端的垃圾回收（GC）机制将保留 Session 文件更长时间，可能会略微增加 `/tmp` 或 Session 存储目录的磁盘占用。
- 考虑到高并发场景，需关注 Session 文件的清理频率。
